# Smart AI CRM — Phase 0 IA/UX Design (первый подход)

**Дата:** 2026-05-03
**Статус:** Draft — первый подход к IA, валидация через wireframe-прототип
**Контекст:** Phase 0 из PRD `Smart AI CRM v1.3` — спроектировать UX до написания кода
**Аудитория спека:** автор, для self-validation IA перед уточнением scope MVP и dev-планом

---

## 1. Цель этого упражнения

Не готовый дизайн. Не финальный UX. **Первый набросок IA**, чтобы:
- увидеть как «AI-флоу» (приоритизация → карточка с обогащением → пайплайн) живёт в реальном интерфейсе
- проверить, что навигация между Daily Plan / Lead Card / Pipeline не разваливается
- получить артефакт для разговоров: «вот что мы строим»

Артефакт — 3 HTML-мокапа в визуальном компаньоне (см. References внизу).

---

## 2. Scope

**В прототипе Phase 0 (3 экрана):**
1. **Daily Plan** — главный экран дня (home)
2. **Lead Card** — карточка с AI Brief как вкладкой по умолчанию
3. **Pipeline / Kanban** — вторичный экран для общего обзора

**Вне scope этого захода:**
- Unified Inbox (только tab-заглушка в карточке)
- Activities timeline
- Дашборд руководителя
- Настройки workspace (воронки, команда, AI-провайдеры)
- Онбординг
- Empty states, loading states, error states
- Mobile / responsive

**Фиделити:** wireframe+ (структурный + минимальный визуальный язык). Не hi-fi.
**Аудитория прототипа:** только автор, для валидации IA. Не для тестирования с менеджерами и не для инвестора.

---

## 3. Ключевое IA-решение: Today-first

Из 3 рассмотренных подходов выбран **C — Today-first / Plan-as-home**:

```
┌──────────┬─────────────────────────────────────┐
│ Sidebar  │ Today (главный экран дня)           │
│  📋 Today│   ↓ click задачу                    │
│  📊 Pipe │ Lead Card (modal/full)              │
│  💬 Inbox│   ↓ или из Pipeline                 │
│  📦 Arch │ Pipeline = вторичный обзор          │
└──────────┴─────────────────────────────────────┘
```

**Почему:**
- Уникальная ценность CRM = AI-приоритизация. Ставим её хозяином экрана, а не Kanban.
- Менеджер открывает CRM → сразу видит «что делать», а не пустой пайплайн.
- Pipeline никуда не девается, но не «стартовая точка».

**Что НЕ выбрано и почему:**
- *A. Классический CRM (sidebar + full-screen Kanban):* клон AmoCRM, не имеет смысла прототипировать.
- *B. Pipeline + Drawer карточки (Linear-style):* хороший паттерн, но AI-флоу не такой выраженный — Kanban перетягивает внимание.

## 4. Спецификации экранов

### 4.1 Daily Plan (home)

**Назначение:** утренний экран менеджера, готовый план дня от AI.

**Структура:**
- **Sidebar** (200px) — Today (active) / Pipeline / Inbox / Архив + список воронок + профиль
- **Шапка:**
  - Приветствие: «Доброе утро, Елена 👋» + дата + N задач
  - Расчёт нагрузки (правый угол): «Доступно 6.5 ч / Загружено 5.2 ч / 80%» + прогресс-бар
- **3 группы задач** с цветовым приоритетом:
  - 🔥 **Срочно** (red, дедлайн / обещание сегодня)
  - ⚡ **Приоритет** (orange, rotting / горячие / новые с высоким fit)
  - 📋 **Плановое** (blue, регулярное продвижение)
- **Footer:** итоговое время + ссылка «пересчитать» + время генерации AI

**Карточка задачи содержит:**
- Название компании · стадия (pill) · сумма+масштаб
- 🤖 AI-подсказка (1-2 предложения, контекстная — почему важно сейчас)
- Время на задачу + причина приоритета (rotting Nд / дедлайн сегодня / создан вчера)

**Что AI-подсказка должна делать (валидировано на примерах):**
- Не пересказывать карточку, а указывать **точку входа сейчас**
- Ссылаться на свежий сигнал (тест с Магнитом, смена ЛПР, движение конкурента)
- Подсказывать конкретного контактного лица где возможно

---

### 4.2 Lead Card (AI Brief)

**Назначение:** одно место для работы с лидом. AI Brief — главная вкладка.

**Структура:**
- **Sidebar** (тот же)
- **Header:** Stars Coffee · стадия · сумма · ответственный · источник · дата + кнопки (Переместить, + Активность, ⋯)
- **4 вкладки:**
  - 📋 Информация (поля + редактирование)
  - 💬 Переписка (Unified Inbox — заглушка в Phase 0)
  - 🤖 **AI Brief** ★ — **default tab**
  - 🕐 Активности (timeline — заглушка)
- **Body** = 2 колонки:
  - **Левая (280px):** компания, сайт, сегмент, **контакты ЛПР как карточки** (Антон Пинский, Эрнесто Гонсалес, Соломатин), теги
  - **Правая:** AI Intelligence Panel (см. ниже)

**AI Intelligence Panel — секции в порядке:**
1. **Score header** — большая цифра (9/10) + одна строка вердикта + время последнего обогащения + ссылка «↻ обновить»
2. **🏢 Профиль компании** — 2-3 предложения с источниками (Forbes / TAdviser / Rusprofile)
3. **📏 Масштаб** — bullet list (точки, города, форматы)
4. **📊 Сигналы** — 2x2 grid:
   - 📈 Зелёные сигналы (рост, расширение, партнёрства)
   - ⚠️ Красные сигналы (риски, что НЕ продавать)
5. **🎯 Подсказки по ЛПР** — кого выбрать как точку входа и почему
6. **🎬 Рекомендованные шаги (3)** — нумерованный список с action-кнопками рядом (📧 Написать / 📋 Создать / 📌 К себе)

**Принцип:** каждая секция отвечает на один вопрос менеджера перед звонком — кто, какой масштаб, что хорошо/плохо, к кому идти, что делать.

---

### 4.3 Pipeline / Kanban

**Назначение:** общий обзор воронки. Не главный экран — открывается когда нужен «вид сверху».

**Структура:**
- **Sidebar** (Pipeline active)
- **Top bar:** Pipeline · переключатель воронки · stats (21 сделок / ₽98M / 3 rotting) · view-switcher (Kanban/List/Focus) · Фильтры · + Лид
- **🤖 AI-баннер «Сигналы дня»** (горизонтальный, 3 карточки) — выводит 3 главных AI-алерта пайплайна (Stars urgent / Магнит rotting / Дринкит новый горячий)
- **Доска** — 6 колонок: Новые лиды · Квалификация · КП отправлено · Переговоры · Согласование · Закрыто (won)
- Каждая колонка: header с названием, count, total суммой, цветной полоской

**Карточка лида в Kanban:**
- Название (жирно) · **Fit Score** (цветной pill: high зелёный / med жёлтый)
- Сумма + краткий масштаб («45 франшиз», «38 точек МСК»)
- **Теги** (срочно / rotting / горячий / смена ЛПР / тендер)
- Footer: время в стадии · аватар ответственного
- **AI-флаг в углу** для критичных: 🔥 (срочно), ⚡ (rotting+конкурент), NEW (только обогащено)

**Stages по умолчанию:** Новые лиды → Квалификация → КП отправлено → Переговоры → Согласование → Закрыто (won/lost). Кастомизируется на воркспейс.

## 5. Связка экранов (state flow)

```
Утро → Today (default) → клик на «Stars Coffee» в группе СРОЧНО
                              ↓
                      Lead Card (AI Brief default)
                              ↓ (← Back to Today)
                      обратно в Today
                              ↓
                      После обработки 5-6 задач → click Pipeline в sidebar
                              ↓
                      Kanban для общего обзора, drag-drop при необходимости
                              ↓ click карточки
                      Lead Card (AI Brief default)
```

Принцип: **Today — это inbox задач**. Pipeline — это **карта**. Карточка — место **работы**. Не три равноправных экрана, а иерархия: «делать → смотреть → редактировать».

## 8. Выбранный путь: A — углубить Phase 0

Стадии:

**Stage 1 — добить wireframe-набор (текущая фаза):**
- Inbox screen (новый, Unified: Email + Telegram)
- Empty state Daily Plan
- Mobile draft (1-2 ключевых экрана)
- Multi-manager Pipeline view
- Drag-drop UX в Kanban (annotated states)
- Modal vs Full-page Lead Card — A/B сравнение

**Stage 2 — переход на hi-fi:**
- Не Figma. Дизайн-систему и hi-fi экраны делаем на основе доступных design-скиллов (frontend-design, ui-design:*, design-systems, design-research).
- Подобрать визуальный язык (likely modern SaaS / Linear-style + DrinkX-аксент).
- Прогон всех экранов через hi-fi.

**Stage 3 — кликабельный прототип:**
- HTML/JS прототип со связанными переходами Today ↔ Card ↔ Pipeline.
- Готов к показу 2-3 менеджерам DrinkX (PRD-критерий Phase 0 завершения).
